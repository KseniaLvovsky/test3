# Instructions

Please read all instructions before starting.

This mock assignment is designed to simulate an early-stage QA screening task focused on:
- Manual QA fundamentals
- API testing concepts
- Test case design
- Bug reporting
- Optional basic scripting

## Rules
- For Task 1 and Task 2, pretend you are not allowed to use the internet.
- Write your answers in the files inside the `answers/` folder.
- Keep answers clear and concise.
- You may use markdown formatting, but readability matters more than formatting.

## What to submit
A completed repository with your answers in:
- `answers/task1_answers.md`
- `answers/task2_answers.md`
- `answers/task3_bug_report.md`
- `answers/task4_optional_notes.md` (optional)
- `answers/task4_optional_solution.py` (optional)

## Recommendation
Do not open the `reference_solution/` folder until you finish all required tasks.
