## Part B — Short answer

1. Verification checks whether the product is built according to requirements; validation checks whether it meets user needs and expected behavior.
2. Severity reflects technical/business impact; priority reflects how urgently the issue should be addressed.
3. Exploratory testing is useful when requirements are unclear, time is short, or you want to quickly discover unexpected risks.
4. Expected result, actual result, environment, severity/priority, evidence such as logs/screenshots, and any relevant test data.
